# API Routes

The backend exposes **30 routes** across five domains: **auth**, **extraction**, **jobs**, **documents**, and **refinement**. All data routes bypass JWT auth using a single canonical system user ‚Äî see [Auth Architecture](#auth-architecture) below.

---

## Auth architecture

Despite `fastapi-users` being configured with JWT auth (`tokenUrl=/auth/jwt/login`), every **data route** (extract, jobs, documents, refine) uses `Depends(get_system_user)` instead of `Depends(current_active_user)`. This dependency returns a single "system" user row (email: `system@triple-h.local`), created lazily on first request. This enables auth-free operation for the single-tenant tool while satisfying database FK constraints.

The `/auth/*` and `/users/*` routes remain fully functional with JWT for anyone who wants them; JWT is simply not checked on data routes.

---

## Middleware (applied to all routes)

| Middleware | Effect |
|---|---|
| `CORSMiddleware` | Allows configured origins (default `*`), credentials, all methods/headers. |
| `TimingMiddleware` | Injects `Server-Timing` response header + logs `METHOD /path -> STATUS in Xms`. |
| OpenTelemetry `FastAPIInstrumentor` | Auto-instruments every route with OpenTelemetry spans. |

## 2. Extraction routes

Source: `app/routes/extract.py`

### `GET /extract/models`

Return the available LLM models for the frontend's upload dropdown.

**Auth**: None
**Response**: `list[ExtractionModelOption]`

Each option includes: `id` (LiteLLM model key), `label` (human-readable), `provider`, `supports_multi_image`, `is_default`, `note`.

Current models:
- `ollama-gemma4-31b` (default) ‚Äî Gemma 4 31B via Ollama Cloud
- `gemma-4-31b` ‚Äî Gemma 4 31B via Google AI Studio
- `groq-llama4-scout` ‚Äî Llama 4 Scout 17B via Groq
- `groq-llama4-maverick` ‚Äî Llama 4 Maverick 17B 128e via Groq
- `nim-llama-90b-vision` ‚Äî Llama 3.2 90B Vision via NVIDIA NIM

---

### `POST /extract`

Chandra OCR only ‚Äî no database persistence. Returns raw OCR chunks and markdown.

**Auth**: None
**Request body** (multipart/form-data): `file` (UploadFile ‚Äî PDF or image)
**Response**: `ExtractResponse` ‚Äî `filename`, `page_count`, `markdown`, `html`, `chunks`, `checkpoint_id`

**Purpose**: Quick preview / debug path. No document record is created in the database.

---

### `POST /extract/structured` ***REDUNDANT DUE TO JOB ROUTES***

Full extraction pipeline: Chandra OCR + PDF page rendering + LLM extraction. Persists a `Document`, `DocumentPage[]`, and `ExtractionRun`.

**Auth**: System user
**Request body** (multipart/form-data):
- `file` (UploadFile) ‚Äî PDF or image
- `doc_type` (str, optional) ‚Äî manual override: `delivery_order`, `weighing_bill`, `invoice`, `petrol_bill`. Leave empty for auto-classification.
- `model` (str, default `ollama-gemma4-31b`) ‚Äî LiteLLM model id
- `dpi` (int, default `150`) ‚Äî PDF rasterization DPI

**Response**: `StructuredExtractResponse` ‚Äî `document_id`, `extraction_run_id`, `is_new_document`, `filename`, `doc_type`, `model`, `page_count`, `duration_ms`, `extracted` (key-value pairs), `checkpoint_id`

**Errors**: 502 on Chandra failure or LLM failure.

---

### `POST /documents/{document_id}/reextract`

Re-run Chandra + LLM extraction for an existing document. Pulls the original PDF from blob storage, runs the full pipeline, and appends a new `ExtractionRun` (the previous run is demoted from `is_current`).

**Auth**: System user
**Path**: `document_id` (UUID)
**Request body**: `ReextractRequest` ‚Äî `{ doc_type?, model?, dpi? }`

**Response**: Same as `POST /extract/structured`

**Idempotent**: Appending a new run never overwrites existing field reviews ‚Äî they remain attached to whichever run they were made on.

## 3. Job (async extraction) routes

Source: `app/routes/jobs.py`

These routes provide an asynchronous alternative to `/extract/structured`. Instead of blocking for 30-200s, the request returns in ~200ms with a `job_id`. A background worker pool processes the job and updates the database. The frontend polls or subscribes to SSE for status updates.

### `POST /extract/jobs`

Queue an extraction job. Returns immediately with a `job_id`.

**Auth**: System user
**Status code**: 202 Accepted
**Request body** (multipart/form-data):
- `file` (UploadFile) ‚Äî PDF or image
- `doc_type` (str, optional)
- `model` (str, default `ollama-gemma4-31b`)
- `dpi` (int, default `150`)
- `Idempotency-Key` (header, optional) ‚Äî client-supplied UUID. Same key + active job = returns existing job, no duplicate work.

**Response**: `SubmitJobResponse` ‚Äî `job_id`, `document_id`, `is_new_document`, `deduped`, `status`, `poll_url`, `stream_url`

**Dedup logic**: If the same `Idempotency-Key` is sent while a prior job is still active (pending/running), the existing job is returned. Without a client key, the server generates one; dedup still works via `(content_hash, model, doc_type)`.

---

### `GET /jobs`

List recent jobs, newest first.

**Auth**: System user
**Query params**: `limit` (int, default 50), `statuses` (str, optional comma-separated filter: `pending,running,succeeded,failed`)

**Response**: `list[JobListItem]` ‚Äî each with `job_id`, `document_id`, `filename`, `model`, `status`, `stage`, `error`, `run_id`, `attempts`, `created_at`, `started_at`, `finished_at`

**Purpose**: Hydrate the frontend queue panel across page refreshes.

---

### `GET /jobs/{job_id}`

Snapshot of a job's current state.

**Auth**: System user
**Path**: `job_id` (UUID)

**Response**: `JobStatusResponse` ‚Äî `status` (pending/running/succeeded/failed), `stage`, `error`, `run_id`, `attempts`, timestamps

**Purpose**: Frontend polls this every 1-2 seconds for status updates.

---

### `GET /jobs/{job_id}/stream`

Server-Sent Events (SSE) feed for a job's lifecycle.

**Auth**: System user
**Path**: `job_id` (UUID)

**Response**: `text/event-stream`

- Initial frame: `retry: 2000` (browser auto-reconnect hint)
- On status/stage change: `event: status` followed by `JobStatusResponse` JSON
- Terminal state: connection closes after final frame
- Timeout (10min): sends `event: timeout` frame and closes

**Implementation**: DB-poll loop at ~500ms cadence. Yields only when the `(status, stage)` tuple changes. Closes when the job enters a terminal state or after 10 minutes.

**Headers**: `Cache-Control: no-cache`, `X-Accel-Buffering: no`, `Connection: keep-alive`

---

### `POST /jobs/{job_id}/cancel`

Flip a pending or running job to `failed` with error `"cancelled by user"`.

**Auth**: System user
**Path**: `job_id` (UUID)

**Response**: `JobStatusResponse`

**Idempotent**: Already-terminal jobs (succeeded/failed) return their current state unchanged with HTTP 200.

**Caveat**: The worker does not yet check this flag mid-pipeline. A running job's pipeline continues until completion; the result is simply dropped since `mark_succeeded` gates on `status='running'`.

## 4. Document routes

Source: `app/routes/documents.py`

Prefix: `/documents`

### `GET /documents`

List every document, newest first (single-tenant ‚Äî no per-user scoping).

**Auth**: None
**Response**: `Page[DocumentSummary]` (paginated via `fastapi-pagination`)

Each summary: `document_id`, `filename`, `mime_type`, `size_bytes`, `page_count`, `doc_type`, `status`, `created_at`.

Supports `?page=&size=` query parameters via `fastapi-pagination`.

---

### `GET /documents/{document_id}`

Full document detail including the current extraction run, page dimensions, and field reviews.

**Auth**: None
**Path**: `document_id` (UUID)

**Response**: `DocumentDetail`
- `document` ‚Äî `DocumentSummary`
- `page_count`
- `pages` ‚Äî list of `PageDims` (page_no, width_px, height_px)
- `current_extraction` ‚Äî `ExtractionRunPayload` or `null`
  - Includes `extracted_view` (KVPs after user-review overlay), `field_pages` (maps each field to its page number), `reviews` (list of `FieldReviewSummary`)

**Field-pages logic**: Two systems are layered to determine which page each field belongs to:
1. Substring matching (`compute_field_pages`) ‚Äî finds LLM-emitted values that match Chandra block text
2. Envelope provenance (`field_pages_from_envelope`) ‚Äî authoritative for regex-anchored fields (do_number, po_number, item descriptions). Envelope wins on overlap.

---

### `PATCH /documents/{document_id}/extraction`

Append `FieldReview` rows for user corrections. Does **not** mutate the extraction payload ‚Äî edits are applied as an overlay at read time.

**Auth**: System user
**Path**: `document_id` (UUID)
**Request body**: `PatchExtractionRequest` ‚Äî `{ edits: [{ field_path, edited_value, remark? }] }`

**Response**: `PatchExtractionResponse` ‚Äî `extraction_run_id`, `applied` (count), `extracted_view` (overlayed result), `reviews`

**Immutability design**: Since edits are stored as separate review rows and overlaid at read time, re-running the pipeline never silently overwrites user corrections. The new run produces a fresh immutable payload; human edits remain attached to whichever run they were made on.

---

### `GET /documents/{document_id}/pages/{page_no}.png`

Return a pre-rendered page PNG from blob storage.

**Auth**: None
**Path**: `document_id` (UUID), `page_no` (int)

**Response**: `image/png`

---

### `GET /documents/{document_id}/pages/{page_no}/blocks`

Return Chandra OCR blocks for a page, scaled to rendered-PNG coordinate space. Includes `field_anchors` for UI overlay highlighting.

**Auth**: None
**Path**: `document_id` (UUID), `page_no` (int)

**Response**: `PageBlocksResponse`
- `page_no`, `width_px`, `height_px`
- `blocks` ‚Äî list of `BlockOverlay` (block_id, block_type, text, quad polygon, bbox)
- `field_anchors` ‚Äî `dict[str, str]` mapping field paths to block IDs

**Purpose**: Frontend paints a transparent overlay over the page image ‚Äî one positioned element per block. When a KVP row is hovered, the matching block lights up.

**Field-anchor logic**: Same dual system as document detail ‚Äî substring matching and envelope provenance are layered, with envelope winning on overlap.

---

### `GET /documents/{document_id}/pages/{page_no}/annotated.png`

Return the page PNG with OCR detection boxes overlaid and a text panel. Composites on demand using PIL ‚Äî no pre-rendering.

**Auth**: None
**Path**: `document_id` (UUID), `page_no` (int)

**Response**: `image/png` with `Cache-Control: private, max-age=300`

---

### `DELETE /documents/{document_id}`

Hard-delete a document and everything it owns (cascade: DocumentPage, ExtractionRun, FieldReview, RefinementRun). Blob cleanup is best-effort.

**Auth**: None
**Path**: `document_id` (UUID)

**Response**: 204 No Content

---

### `DELETE /documents/{document_id}/reviews/{review_id}`

Delete a single `FieldReview` row from the audit log. Validates the review belongs to the specified document.

**Auth**: None
**Path**: `document_id` (UUID), `review_id` (int)

**Response**: 204 No Content

---

## 5. Refinement routes

Source: `app/routes/refine.py`

### `POST /refine/{extraction_run_id}`

Run one VLM refinement pass over the most relevant page of an extraction's DoclingDocument scaffold. Persists a new `refinement_run` row, demotes the prior `is_current` row.

**Phase 1 scope**: Classification only ‚Äî `assign` and `reject` patches. Geometry-edit ops (`add`, `move`) deferred to later phases.

**Auth**: System user
**Path**: `extraction_run_id` (int)
**Query params**: `page_no` (int, ge=1, default=1) ‚Äî 1-indexed page number

**Response**: `RefineResponse`
- `refinement_run_id`, `extraction_run_id`
- `vlm_model`, `prompt_version`
- `duration_ms`, `token_usage`
- `result` ‚Äî `RefinementResult` with patches + ARQ trace
- `scaffold_out` ‚Äî the patched DoclingDocument JSON

**Errors**: 404 if extraction run or page not found, 409 if scaffold is missing or no field keys exist, 502 on VLM failure.
